Pre-Migration Setup:

This scenerio utilizes the CSM setting to start a session in GC mode. This is useful when the need arises for keeping steady state Global Mirror running but replacing a new array. The steps outlined involves adding a GC session off the GM secondarys. Once new GC session is created at migration it will be converted to a GM session replacing the existing. 

Prerequisites:

  • Running Global Mirror session
  • A staged Global Copy session where the primary is the I2s from running Global Mirror session
  • A staged Global Mirror session where the new primary is the new array, which is the GC secondaries for the above staged Global Copy session
  • Entry in the rmserver.properities file for nocopy=true for the new Global MIrror session. 
    • com.ibm.csm.session_name.role_pair_name.nocopy=true - where session_name is the name of the session and role_pair_name is the name of the rolepair
    • Example:
      • com.ibm.csm.My_MGM.H1-H3.nocopy=true - where My_MGM is the name of the session and H1-H3 is the role pair that I want established with no copy.
      •  ** Replace all spaces with "_" in session name.  Property is case sensitive.

Procedure:

 * Production Global Mirror and the staged Global Copy  session should be running. Global Copy should be in state of prepared and 100%. Global Mirror should be prepared state & normal. 

 ** HOST IO should be quiesced to ensure consistency. 

  1. Suspend GM session
  2. Stop when Drained for the Global Copy session
  3. Terminate GM session 
  4. Recover Global Copy session
  5. Start  H1 → H2 for new GM session 
  6. Wait for 100% progress. Should be a NoCopy since the property of the session was added to the CSM properties file. Session will NOT go prepared until steps below are complete. 
  7. Remove Global Copy session
  8. Suspend new Global Mirror session
  9. Start H1 -→ H2 for new GM session. This will remove the bidirectional replication for each copy set just leaving the desired direction. It will also allow the session to start taking CGs for session. 
  10. Remove the nocopy=true in the properties file. VERY VERY IMPORTANT
